# Security Policy

## Supported Versions

This repository is an early research scaffold. Security fixes apply to the current `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for vulnerabilities involving credentials, private data, or restricted source material.

Report concerns privately to the repository owner through GitHub. Include:

- affected files or workflows
- steps to reproduce, if applicable
- expected impact
- suggested mitigation, if known

## Research Data Caution

Do not commit private notes, restricted PDFs, paywalled texts, API keys, credentials, or unlicensed corpora. If sensitive or restricted material is committed accidentally, notify the maintainer immediately so it can be removed from the repository history if necessary.
